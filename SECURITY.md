# Security Policy

TTP is security-critical infrastructure.

We welcome responsible disclosure and external review.

---

## Reporting Vulnerabilities

Please report privately to maintainers.

Include:
- description
- reproduction steps
- potential impact

Do not open public issues for vulnerabilities.

---

## Scope

Security-sensitive areas:

- token validation
- signature handling
- aggregation logic
- receipt replay protection
- issuer trust assumptions

---

## Response

We aim to:
- acknowledge within 48 hours
- triage severity
- coordinate disclosure
- release fixes quickly

---

## Philosophy

- minimize state
- minimize lifetime
- verify everything
- assume adversaries
